## Migrate — Vault Structure Migration

Port a pre-separation vault backup into the current `pulse-vault/` structure. This skill automates what would otherwise be a manual copy-and-transform operation.

**Not for Neo4j graph migration.** Use `/migrate-graph` for that.

---

### Sense — Validate Inputs

1. **Require a backup path argument.** If none given:
   ```
   Usage: /migrate <path-to-backup>
   Example: /migrate ../vault-backup-2026-05-30

   The backup should be a copy of your old vault directory (before the engine/vault separation).
   ```
   Stop here.

2. **Resolve the path** relative to the repo root if not absolute.

3. **Verify the backup exists** and has at least one of: `Maps/`, `Notes/`, `Daily/`, `Inbox/`, `Sati/`.
   If none found, report: "No recognizable vault structure found at `<path>`. Expected at least one of Maps/, Notes/, Daily/, Inbox/, Sati/."

4. **Inventory the backup:**
   - Count `Maps/*.md` files (excluding `INDEX.md`, `[INIT]*.md`, `_system/` subdir)
   - Count `Notes/*.md` files (excluding `.keep`)
   - Count `Daily/*.md` files
   - Count `Daily/logs/*.md` and `Daily/cache/*.json`
   - Count `Inbox/*.md` files (excluding `.keep` and `multi-agents/`)
   - Check for `Sati/emergence-log.md`
   - Check for `EVOLUTION.md` at backup root
   - Check for `user.config.yaml` at backup root
   - Check for `CLAUDE.md` at backup root (look for `<!-- SLUG-TABLE-START -->` block)

5. **Read effort metadata from backup Maps** — for each `Maps/*.md` (excluding INDEX, INIT, _system):
   - Extract frontmatter: `effort`, `context_batch`, `base_priority`, `purpose`, `aliases`
   - Note if `Maps/Pulse.md` exists at root (old system map location — needs to move to `_system/`)

6. **Check if backup has `user.config.yaml`** — if yes, it's already in new format; use directly.
   If no, synthesize from Map frontmatter (see Act step 9).

7. **Check current `pulse-vault/Maps/`** for already-populated user Maps (excluding `[INIT]` templates and `_system/`). If found, warn: "pulse-vault already has Maps. Migration will overwrite them. Existing files will be replaced."

---

### Surface — Present Migration Plan

Show the plan and ask for confirmation. Example:

```
## Migration Plan

**Source:** ../vault-backup-2026-05-30

**Files to copy:**
- Maps: 10 user Maps + Pulse.md → _system/
- Notes: 12 files
- Daily: 3 session files, 3 logs, 1 cache file
- Inbox: 0 captures
- Sati: emergence-log.md

**Efforts detected:** work, studia, dharma, relationships, zdrowie, maintenance, judo, prawo-jazdy, zew-cthulhu, personal

**user.config.yaml:** will be synthesized from Map frontmatter (no existing config found)

**Cleanup:** [INIT] Work.md, [INIT] Personal.md, [INIT] Health.md will be removed (conflict with ported Maps)

**EVOLUTION.md:** will be copied to pulse-vault/EVOLUTION.md

Proceed with migration? (yes/no)
```

Wait for confirmation before Act.

---

### Act — Execute Migration

Run these steps in order. Report each step as it completes.

**1. Copy Maps:**
```bash
cp <backup>/Maps/*.md pulse-vault/Maps/ 2>/dev/null || true
# Skip INDEX.md — it will be regenerated by pulse-calc.py
rm -f pulse-vault/Maps/INDEX.md
```

**2. Handle old system map** (`Maps/Pulse.md` at backup root → `_system/`):
```bash
if [ -f "<backup>/Maps/Pulse.md" ]; then
  cp <backup>/Maps/Pulse.md pulse-vault/Maps/_system/Pulse.md
  rm -f pulse-vault/Maps/Pulse.md  # remove the copy that went to Maps/ in step 1
fi
```

**3. Copy Notes:**
```bash
cp <backup>/Notes/*.md pulse-vault/Notes/ 2>/dev/null || true
# Preserve archive/ if it has content
cp <backup>/Notes/archive/*.md pulse-vault/Notes/archive/ 2>/dev/null || true
```

**4. Copy Daily files:**
```bash
cp <backup>/Daily/*.md pulse-vault/Daily/ 2>/dev/null || true
cp <backup>/Daily/logs/*.md pulse-vault/Daily/logs/ 2>/dev/null || true
cp <backup>/Daily/cache/*.json pulse-vault/Daily/cache/ 2>/dev/null || true
```

**5. Copy Inbox captures** (triaged: false items):
```bash
cp <backup>/Inbox/*.md pulse-vault/Inbox/ 2>/dev/null || true
```

**6. Copy Sati:**
```bash
cp <backup>/Sati/*.md pulse-vault/Sati/ 2>/dev/null || true
```

**7. Copy EVOLUTION.md** (if present in backup):
```bash
cp <backup>/EVOLUTION.md pulse-vault/EVOLUTION.md 2>/dev/null || true
```

**8. Remove conflicting [INIT] template Maps** — any `[INIT] *.md` in `pulse-vault/Maps/` whose `effort:` slug matches a ported Map:
- Read each `[INIT]` Map's frontmatter for its `effort:` slug
- If a non-INIT Map with the same slug now exists in `pulse-vault/Maps/`, delete the `[INIT]` version
- Report which were removed

**9. Write `pulse-vault/user.config.yaml`:**
- If backup has `user.config.yaml`: copy it directly
- If backup has no `user.config.yaml` but has `CLAUDE.md` with a slug table (`<!-- SLUG-TABLE-START -->`...`<!-- SLUG-TABLE-END -->`): parse the table for slug, batch, and aliases; combine with Map frontmatter for base_priority and purpose; synthesize user.config.yaml
- If neither: synthesize entirely from Map frontmatter
- Always exclude `pulse` slug (system-reserved)

Synthesized `user.config.yaml` format:
```yaml
# Synthesized by /migrate from vault backup — review and adjust as needed.

efforts:
  - slug: <slug>
    batch: <context_batch from Map>
    base_priority: <base_priority from Map>
    purpose: "<purpose from Map>"
    aliases: [<aliases from Map>]
  ...

batches:
  - name: <batch-name>
    shared_context: "<derived or placeholder>"
    mindset: "<derived or placeholder>"
  ...
```

If any batch in the Map frontmatter isn't one of the four defaults (Work, Projects, Maintenance, Leisure), include it in `batches:` with a placeholder `shared_context` and `mindset` and a comment: `# TODO: fill in shared_context and mindset`.

**10. Port user overrides from backup CLAUDE.md to `pulse-vault/CLAUDE.md`:**
- If backup `CLAUDE.md` has an external_input cadence table or other user-specific overrides (anything NOT in the engine CLAUDE.md conventions): port those sections to `pulse-vault/CLAUDE.md` under `## User Notes`
- Do not copy engine conventions — those are already in `pulse-engine/CLAUDE.md`

**11. Validate with pulse-calc.py:**
```bash
python3 pulse-engine/scripts/pulse-calc.py --vault pulse-vault
```
- If exit 0: report "Validation passed. N efforts loaded, 0 warnings."
- If exit nonzero: show the error and list what may need manual attention
