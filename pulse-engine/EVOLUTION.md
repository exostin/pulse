# PULSE Engine Evolution Log

> Engine change history. Record significant architecture, formula, skill protocol, or schema changes here.
> Update this file *first* when the engine evolves, then propagate to `SYSTEM.md`, `CLAUDE.md`, `README.md`, and skills.
> For user vault configuration changes (effort additions, batch tweaks, calibration), see `pulse-vault/EVOLUTION.md`.

| Date | Change | Reason |
|------|--------|--------|
| 2026-03-12 | Redesign: auto-triage, reflective review, defrag | Triage drops confirmation loops, review becomes pure reflection, new /defrag skill absorbs all bookkeeping. Separates thinking (human) from filing (agent). |
| 2026-03-12 | Two-axis context batching with soft suppression | Batches now defined by shared_context (domain) and mindset (cognitive mode). Low-value batches are soft-suppressed in daily checklists and pulse briefings. Batch definitions live in CLAUDE.md. |
| 2026-03-12 | Compact pulse briefing with fold-line suppression | Pulse now shows a compact view by default — batch gating + effort-level suppression collapse low-signal items into a fold-line. Full landscape on request ("unfold"). |
| 2026-03-12 | Conversational Daily Note flow + defrag logging | Daily Note now accretes through /pulse conversation (not batch-generated). /defrag logs to Daily Note. /birdseyereview repositioned as periodic audit. /focus clarified as deep flow tool. |
| 2026-03-12 | Background sub-agent capture | /capture now delegates to a background agent for zero context disruption. Main conversation flow is preserved. |
| 2026-03-12 | /efforts skill + pre-built default Maps | Engine ships with default Maps. /efforts skill handles add, splinter, merge, review with litmus test and anti-spaghetti guidelines. |
| 2026-03-12 | Eliminated efforts.yaml | Maps are the sole source of truth for effort definitions. Context batch definitions moved to CLAUDE.md. Pre-built default Maps replace YAML bootstrap. |
| 2026-03-12 | Replace effort_estimate with effort_level + timescale | effort_level (trivial/small/medium/large) captures mental absorption. timescale (daily→annual) captures periodicity and addresses the 7-day recency bias for longer-cadence items. |
| 2026-03-12 | Session Log — decision trace layer for debugging | Replaced one-liner defrag log with per-decision traces. Added persistent logging to /recompute, /defrag, /pulse, and /triage. All append to Session Log. |
| 2026-03-12 | Unify schema: `domain:` → `effort:` | Personal and public engine diverged — one used `domain:`, the other `effort:`. Unified on `effort:` everywhere. Separate efforts can live under the same problem domain; "effort" is the more precise term. |
| 2026-03-14 | Priority feedback loop & trust calibration | Added calibration log (`Notes/pulse-priority-calibration.md`), Minor Actions sections in Maps, fuzzy item detection, validation prompt in /pulse, session-end recompute in /close, Minor Actions cleanup in /defrag. Introduced PAR metric and Phase 1→2 trust transition criteria. |
| 2026-03-14 | close_complete flag — skip redundant /pulse startup | `/close` sets `close_complete: true` in Daily Note frontmatter after successful defrag+recompute. Next `/pulse` reads yesterday's flag to skip light defrag and inline recompute, trusting cached weights. Inbox scan still runs for overnight captures. Reduces morning startup latency with no accuracy loss. |
| 2026-03-14 | Replace effort_factor with importance-weighted loop_factor | `effort_factor` (note update frequency) overlapped with `recency_boost` — both measured 7-day activity. Replaced with `loop_factor`: per-item importance weighting (high +0.04, medium +0.02, low +0.01, cap 0.10). Added `importance` field to Notes and Minor Actions. Minor Actions now count toward `open_loops`. Plain-text bullets promoted to Minor Actions during defrag. `/pulse` Focus reworked to item-driven (top 5 by importance across all efforts), with importance override for suppression. |
| 2026-05-29 | SRSA framework as system spine | Adopted Sense → Route → Surface → Act as the organizing frame across agent, user, and dyad perspectives. Skills use SRSA phase headers; `srsa:` frontmatter classifies each skill's primary function. Sense/Act written as enumerated logic (always-execute); Route/Surface carry judgment. SYSTEM.md §2 documents all three perspectives. |
| 2026-05-29 | Computation extracted to `scripts/pulse-calc.py` | Priority weights and effective item scores now computed by a deterministic Python script that reads vault frontmatter and emits JSON. The script owns the math; the agent owns judgment (fuzzy flags, suppression reasoning, display). Computation cache in `Daily/cache/` enables cross-day drift detection. |
| 2026-05-29 | Uncapped priority formula + decay recency | `priority_weight` is now an uncapped sum (top-tier ordering fidelity over bounded range). `recency_boost` switched to linear decay from `last_active` (`max(0, 0.12 × (1 − days_since/7))`), reading Map frontmatter directly instead of scanning Daily notes. `loop_factor` uncapped — high loop counts are diagnostic. |
| 2026-05-29 | effective_item_score for Important Items | Item-level ranking (effort weight + importance modifier + due-proximity + status modifier), computed ephemerally during /pulse. Importance is a soft seed, not a display gate. Display threshold ≥0.55, floor 3 / ceiling 20, per-effort cap 3 on high/medium. Two-section display: Important Items + Between Tasks. |
| 2026-05-29 | Waiting-item exception + escalation | Waiting Notes with a due date no longer spike urgency unless within 1 day of due or overdue ("on hold until date" ≠ urgent). Waiting items escalate at 3 days (informational). Minor Actions support `[w]` waiting checkbox with `waiting_since:`. |
| 2026-05-29 | Dependencies & relations on Notes | Added `depends:` (directional) and `related:` (undirected) Note fields, plus inline `depends::` on Minor Actions (Dataview-queryable). `/defrag` surfaces items as unblocked when a dependency target is done/archived. |
| 2026-05-29 | Loop counting boundary | Only Notes under `## Active Threads` (and all Minor Actions) count toward `loop_factor`/`open_loops`. Sub-theme notes (`### Theory & Reference Docs`, etc.) and `subtype: reference` notes are excluded. Map structure is the source of truth for what is an "open loop." |
| 2026-05-29 | `docs/` for perennial reference | Added `docs/` for design principles, theoretical frames, and references that *inform* work without being work. Notes graduate to docs when they become persistent reference. Docs declare `informs:` and appear under a Map's `### Theory & Reference Docs` sub-theme. |
| 2026-05-29 | Sati emergence layer | Added `Sati/` and the emergence log for noticing novel connections, category violations, and reification. Sati is continuous meta-Sense across all nodes; observations logged to session log and `Sati/emergence-log.md`, promoted at 3+ recurrences (human-decided). |
| 2026-05-29 | Session start reads `Maps/INDEX.md` | Session start now reads the precomputed `Maps/INDEX.md` table (agent-readable landscape) instead of `Home.md` (whose Dataview blocks only render in Obsidian). `Home.md` remains the human-facing dashboard. |
| 2026-05-29 | Evolution Log extracted to EVOLUTION.md | Change history moved out of SYSTEM.md §14 into this dedicated file, updated first when the system evolves. |
| 2026-05-30 | Engine/vault separation for open-source redistribution | `pulse-engine/` extracted as a standalone engine package; `pulse-vault/` is now the user's private vault. `user.config.yaml` introduced as the effort/batch configuration file (replaces slug table in CLAUDE.md). `Maps/_system/` introduced for system-type Maps (Pulse.md moved here). Engine conventions in `pulse-engine/CLAUDE.md`; user overrides in `pulse-vault/CLAUDE.md`. `/migrate` skill added for automated vault structure migration. |
